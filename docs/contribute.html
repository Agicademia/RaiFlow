Add a Compliance Check

New checks go in raiflow/gate.py as a _check_* method. Each check returns a CheckResult and never raises. See the design docs for the dual static/semantic pattern.

Write Tests

We use pytest for unit tests and Hypothesis for property-based tests. All new checks need both. Check tests/ for examples.

Development Setup

terminal
# Clone and set up
git clone https://github.com/Agicademia/RaiFlow.git
cd RaiFlow
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Run the full test suite
python -m pytest tests/ -v

# Run with mock LLM (no Ollama needed)
RAIFLOW_MOCK_LLM=true python -m pytest tests/ -v

# Run the compliance gate on itself
raiflow check --stage ci --no-dashboard

Project Structure

raiflow/
├── cli.py              # Click CLI entry point
├── gate.py             # CheckRunner — static + semantic compliance checks
├── manifest.py         # raiflow.yaml Pydantic schema and loader
├── dashboard_server.py # FastAPI + SSE streaming server
├── scanner.py          # AI framework detection
├── scaffolder.py       # raiflow.yaml + workflow generation
├── reporter.py         # JSON compliance report builder
├── generator.py        # pytest file generator from policy YAML
├── evaluators/         # LLM-based EU AI Act evaluators
├── dashboard/          # Single-file SPA (index.html)
└── data/               # Bundled policy YAML + GitHub Actions template

tests/
├── test_gate.py                    # CheckRunner unit tests
├── test_semantic_evaluators.py     # Semantic check unit tests
├── test_reporter.py / test_manifest.py / test_cli.py
├── test_*_property.py              # Hypothesis property-based tests
└── eu_ai_act_test.py               # End-to-end evaluator tests
